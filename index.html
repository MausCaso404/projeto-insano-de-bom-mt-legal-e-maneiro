Sistema de recomendação de filmes por IA

O projeto consiste em uma aplicação de recomendação de filmes e séries personalizada, desenvolvida com Python, utilizando técnicas de Machine Learning e Filtragem Colaborativa. O objetivo é oferecer aos usuários sugestões de conteúdo alinhadas com seus interesses, com base em seu histórico de visualizações e preferências, aprimorando a experiência de uso e aumentando o engajamento com a plataforma.

O sistema é dividido em dois componentes principais: o modelo de recomendação e a interface de interação. O modelo de recomendação é construído com a utilização de algoritmos de Filtragem Colaborativa e Filtragem Baseada em Conteúdo. A Filtragem Colaborativa analisa os padrões de comportamento de usuários semelhantes, enquanto a Filtragem Baseada em Conteúdo avalia as características dos itens assistidos para sugerir opções com base em atributos como gênero, atores e diretor.

Para a implementação do modelo, a linguagem de programação Python é utilizada devido à sua vasta gama de bibliotecas para manipulação de dados e aprendizado de máquina, como Pandas e NumPy. A Scikit-learn é usada para construir os algoritmos de aprendizado de máquina, enquanto o TensorFlow é explorado para implementar modelos de deep learning (caso necessário para melhorar as previsões). Os dados de usuários e preferências são armazenados em um banco de dados PostgreSQL, que permite consultas rápidas e eficientes.

A interface de interação com o usuário é desenvolvida em Flask, criando uma API RESTful que conecta o backend (o modelo de recomendação) com o frontend da plataforma. A aplicação web, consumindo essa API, pode exibir as recomendações personalizadas de maneira dinâmica, atualizando as sugestões conforme o histórico do usuário.

Aplicativo de gestão financeira

O projeto consiste em um aplicativo web de gestão financeira pessoal, desenvolvido com React para o front-end e Node.js/Express para o back-end. O objetivo principal é ajudar os usuários a controlarem suas finanças pessoais de maneira simples e eficiente, permitindo que registrem despesas, acompanhem receitas e gerem relatórios financeiros detalhados para facilitar o planejamento orçamentário.

A interface do aplicativo é projetada com React, oferecendo uma experiência interativa e responsiva. O layout foi pensado para ser intuitivo, com seções dedicadas ao registro de receitas e despesas, visualização de gráficos de fluxo de caixa, e geração de relatórios mensais. O uso de Redux foi implementado para gerenciar o estado global da aplicação, garantindo que as alterações feitas em uma parte do sistema sejam refletidas em tempo real em outras seções, como a atualização do saldo total ou o status de alertas financeiros.

O back-end da aplicação é desenvolvido com Node.js e Express, fornecendo uma API RESTful que permite ao front-end realizar operações de leitura e escrita nos dados financeiros dos usuários. As transações são armazenadas em um banco de dados MongoDB, escolhido por sua flexibilidade e escalabilidade para gerenciar registros financeiros, que incluem campos como descrição, categoria (alimentação, transporte, lazer, etc.), valor e data.

O aplicativo permite que os usuários categorizem suas despesas e receitas, ajudando a identificar padrões de consumo. Para tornar a visualização mais clara e interativa, gráficos são gerados utilizando a biblioteca Chart.js, proporcionando ao usuário uma visão detalhada de sua evolução financeira ao longo do tempo, como um gráfico de barras comparando as despesas de cada mês ou um gráfico de pizza que mostra a distribuição das categorias de gastos.

API voltada ao ecommerce

O projeto consiste em uma API RESTful desenvolvida para uma plataforma de e-commerce, com o objetivo de gerenciar produtos, usuários, pedidos e transações de forma eficiente e escalável. A API foi construída com Node.js e Express no back-end, utilizando MongoDB como banco de dados NoSQL para armazenar informações relacionadas a produtos, usuários e pedidos.

A API tem como objetivo fornecer endpoints para todas as funcionalidades essenciais de uma plataforma de e-commerce, como cadastro e autenticação de usuários, gerenciamento de produtos, processamento de pedidos e integração com sistemas de pagamento. As operações são estruturadas de forma que a plataforma de front-end possa consumir os dados de maneira simples e rápida, proporcionando uma experiência de compra fluida e sem interrupções.

Além disso, a API permite a gestão de produtos, oferecendo endpoints para criar, atualizar, listar e remover itens do catálogo da loja. Cada produto possui atributos como nome, descrição, preço, categoria, estoque disponível e imagens, e a API também oferece filtros de pesquisa para facilitar a busca por categoria, faixa de preço ou nome.
